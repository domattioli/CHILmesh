## Codebase Context

### Current Adjacency Model
- **EdgeMap:** Hash table (min(v0, v1), max(v0, v1)) → edge_id. O(1) lookup.
- **Vert2Edge/Vert2Elem:** Dict[int, Set[int]] for sparse adjacencies.
- **Edge2Vert/Elem2Edge:** ndarray for dense connectivity.
- **Files:** `src/chilmesh/mesh_topology.py` (EdgeMap), `src/chilmesh/CHILmesh.py` (_build_adjacencies).

### Optimization Targets (from CONCERNS.md)
- **Skeletonization bottleneck:** `_skeletonize()` (3.26s = 98% of init time). DCEL offers boundary-peeling efficiency vs. dict lookups.
- **Vert2Edge traversal:** Current O(1) access but high constant (set iteration). DCEL half-edge chains eliminate iteration.
- **Memory footprint:** Current model uses 5 dict structures + 3 ndarrays. DCEL consolidates to 1 edge-centric structure (potentially more compact).
