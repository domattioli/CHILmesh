<objective>
Implement half-edge (DCEL) data structure as an alternative topology representation and establish whether it delivers performance gains over current EdgeMap/dict adjacency model.

**Purpose:** 
Phase v5.0 is a feasibility study, not production adoption. We need empirical data on whether half-edge's efficiency in boundary traversal outweighs the overhead of building a more complex structure. The current EdgeMap + dict model is already O(1) for edge lookup; half-edge's advantage lies in chain-based neighbor iteration vs. set iteration.

**Output:** 
- Prototype half-edge module (internal, swappable)
- Passing test suite on both implementations
- Benchmark table in docs/BENCHMARK.md
- Decision data for Phase 6 adoption gate
</objective>

Half-edge must provide the same contract:
```python
class HalfEdgeTopology:
    def edge2vert(self, edge_id: int) -> Tuple[int, int]: 
        """Return (v1, v2) endpoints."""
    
    def elem2edge(self, elem_id: int) -> List[int]:
        """Return incident edge IDs for element."""
    
    def vert2edge(self, vert_id: int) -> Set[int]:
        """Return incident edge IDs for vertex."""
    
    def vert2elem(self, vert_id: int) -> Set[int]:
        """Return incident element IDs for vertex."""
    
    def edge2elem(self, edge_id: int) -> Tuple[int, int]:
        """Return (elem_left, elem_right), -1 if boundary."""

    # Half-edge specific: chain iteration
    def next_half_edge(self, half_edge_id: int) -> int:
        """Return next half-edge around same element."""
    
    def opposite_half_edge(self, half_edge_id: int) -> int:
        """Return opposite half-edge on adjacent element."""
    
    def twin(self, half_edge_id: int) -> int:
        """Alias for opposite_half_edge."""
```

  <action>
Create a new internal module `src/chilmesh/mesh_topology_halfedge.py` implementing a half-edge (DCEL) data structure. The module must:

1. **Define HalfEdgeTopology class** (~250 lines):
   - Store half-edges as an array of half-edge records (half_edge_id → {next, opposite, vertex, element, edge_id})
   - Build from connectivity_list and Edge2Vert during initialization
   - Implement public accessors matching current EdgeMap/dict interface (per interface block above):
     * `edge2vert(edge_id)` → (v1, v2) ndarray
     * `elem2edge(elem_id)` → List[edge_ids]
     * `vert2edge(vert_id)` → Set[edge_ids]
     * `vert2elem(vert_id)` → Set[elem_ids]
     * `edge2elem(edge_id)` → (elem_left, elem_right)
   - Implement half-edge-specific accessors (for skeletonization optimization in later phases):
     * `next_half_edge(he_id)` → next half-edge around same element
     * `opposite_half_edge(he_id)` → opposite (twin) half-edge
   - Add `@property` accessors to mimic adjacencies dict keys (e.g., `.edge2vert`, `.vert2edge`)

2. **Implement factory function** (per CONTEXT decision D-03):
   - `build_halfedge_from_connectivity(connectivity_list, points, n_verts)` → HalfEdgeTopology
   - Parse element connectivity into half-edges (2 per directed edge, but stored efficiently)
   - Build cross-links (next, opposite, vertex backpointers)
   - Return populated HalfEdgeTopology instance

3. **Design decisions** (per CONTEXT):
   - **Half-edge representation:** 
     * Each directed edge (v0→v1) is one half-edge
     * Each undirected edge has 2 half-edges (one per direction)
     * For n_edges undirected edges, maintain 2*n_edges half-edge records
   - **Storage:** Use numpy arrays for half-edge data (half_edge_id → next, vertex, element, edge_id), dicts for Vert2Edge/Vert2Elem (same as current)
   - **Boundary handling:** Half-edges on boundary have opposite_half_edge = -1; edge2elem returns -1 for boundary
   - **Memory vs. speed:** Accept slightly higher init cost to enable faster skeletonization iteration in Phase 6 (boundary chain traversal)

4. **No public API changes:**
   - HalfEdgeTopology is internal; CHILmesh must not expose it in public methods
   - All accessor methods return same types as current model (ndarrays, sets, tuples)

5. **Performance instrumentation:**
   - Add timing debug lines (print-based, gated by DEBUG env var) for build phases
   - Record "Time to build half-edges: X.XXs" as checkpoint for Phase 6 profiling
