Implement in `src/chilmesh_core/queries.rs`:

**Function: pub fn compute_signed_areas(points: &Array2<f64>, connectivity: &Array2<i32>) -> Array1<f64>**

Algorithm (shoelace formula):
1. For each element in connectivity:
   - Vertices: [v0, v1, v2, v3] (v3 = v0 for triangles)
   - Use only non-padding vertices (triangles: v0, v1, v2)
   - Signed area = 0.5 * |sum of (x_i * (y_{i+1} - y_{i-1}))|
   - Formula: `0.5 * ((x1*y2 - x2*y1) + (x2*y3 - x3*y2) + ... )`
2. Store areas in Array1<f64> [n_elems]
3. Return array

**Function: pub fn ensure_ccw_orientation(points: &mut Array2<f64>, connectivity: &mut Array2<i32>)**

Algorithm:
1. For each element, compute signed area
2. If area < 0 (CW orientation):
   - Reverse vertex order (except padding) to flip sign
   - Update connectivity row
3. Validate all areas positive after reorientation

Rationale: Shoelace is O(n) and accurate for arbitrary polygons. CCW orientation required per SPEC.md. f64 precision necessary for large meshes (f32 would lose precision on Block_O, ~1 million vertices).
