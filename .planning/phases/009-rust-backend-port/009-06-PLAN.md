**Function: pub fn write_fort14(mesh: &RustMesh, path: &str) -> Result<(), RustMeshError>**

Implementation (from Plan 01 stub):
1. Write header: "NE NP" where NE = mesh.n_elems, NP = mesh.n_verts
2. Write elements:
   - For each row in connectivity:
     - Determine ITYPE from elem_type: 3 for triangle, 4 for quad
     - Format: "{elem_id} {ITYPE} {v0} {v1} {v2} [{v3}]" (1-indexed vertices)
     - Write all 3 or 4 vertices based on ITYPE
3. Write coordinates:
   - For each row in points: format "{x:.8e} {y:.8e}"
4. Write to file; wrap errors in RustMeshError::IOError

**Function: pub fn parse_2dm(path: &str) -> Result<RustMesh, RustMeshError>**

Algorithm (2dm format, simpler than Fort.14):
1. Parse header: "E2T" or similar marker
2. Parse element lines: "E {ITYPE} {v1} {v2} {v3} [{v4}]"
   - ITYPE: 3 = triangle, 4 = quad
   - Store in connectivity array [n_elems, 4] with padding
3. Parse node lines: "ND {x} {y}"
   - Store in points array [n_verts, 2]
4. Return RustMesh { points, connectivity, elem_type, n_verts, n_elems }

Rationale: Fort.14 writer completes I/O roundtrip (read/write). 2dm reader extends format support per SPEC.md scope. Both use existing error handling (RustMeshError).

```python
    from chilmesh_core import RustMesh
    
    # Store Rust mesh handle

    self._filepath = filepath
    # Store CRS and other metadata (unchanged)

```

In all query methods:
```python
def _build_adjacencies(self):
    # Delegate to Rust
    edge2vert = self._rust_mesh.get_edge2vert()
    elem2edge = self._rust_mesh.get_elem2edge()
    # ... store in self.adjacencies dict (unchanged)

def signed_area(self):
    return self._rust_mesh.get_signed_areas()

def _skeletonize_medial_axis(self, ...):
    return self._rust_mesh.skeletonize()
```

**Key principle:** Python wrapper holds no business logic. It only delegates to Rust and marshals types (ndarray ↔ Python dict, etc.). All 439 existing tests call Python methods unchanged; tests automatically use Rust backend via wrapper delegation.

**Validation:** No changes to method signatures; backward compatibility guaranteed by design.

Rationale: CONTEXT.md Decision 6 specifies "pure delegation." This ensures single-source-of-truth (Rust) while maintaining Python API surface. Executor reads current CHILmesh.py, identifies delegation points, wraps each in _rust_mesh call.
