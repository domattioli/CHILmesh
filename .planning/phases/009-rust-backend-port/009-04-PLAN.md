**Data Structure: Layer**
```rust
pub struct Layer {
    pub oe: Vec<usize>,        // Oriented elements (boundary)
    pub ie: Vec<usize>,        // Inner elements
    pub ov: Vec<usize>,        // Outer vertices
    pub iv: Vec<usize>,        // Inner vertices
}
```

**Function: pub fn skeletonize_medial_axis(mesh: &mut RustMesh, quality_threshold: Option<f64>) -> Result<Vec<Layer>, RustMeshError>**

Algorithm (layer-by-layer removal):
1. Initialize tracking:
   - remaining_elems: Set<usize> = all element IDs
   - remaining_verts: Set<usize> = all vertex IDs
   - layers: Vec<Layer> = empty

2. While remaining_elems not empty:
   a. Identify boundary elements (edges with ≤1 adjacent element):
      - For each edge in edge2elem:
        - If edge has 1 adjacent element: element is boundary
   b. Mark boundary elements as OE (oriented element)
   c. Identify outer vertices (incident to ≥1 boundary edge):
      - For each boundary edge: mark both endpoints as OV
   d. Identify inner elements (no boundary edges):
      - IE = remaining_elems - OE
   e. Identify inner vertices (not OV):
      - IV = remaining_verts - OV
   f. Create Layer { oe, ie, ov, iv }
   g. Remove boundary elements: remaining_elems -= oe
   h. Remove isolated vertices: remaining_verts -= orphaned verts
   i. Recompute adjacency (edge2elem, connectivity) for next iteration
   j. Optional: pruning by quality threshold (skip layers with median_quality < threshold)

3. Return layers: Vec<Layer>

**Invariants to maintain:**
- len(oe) + len(ie) = n_elems_this_layer
- Sum across all layers: sum(oe + ie) = original n_elems
- OE disjoint from IE
- OV disjoint from IV
- All elements classified exactly once across layers

Rationale: Layer-by-layer removal is O(n) per layer × L layers. Adjacency recomputation is expensive (~O(n)) per layer but necessary for correctness. RESEARCH.md Pitfall 4 emphasizes these invariants; violations indicate bugs.
